2.3 Quaternion Tare

When CMD_CALIB is received during streaming, the slave captures the current orientation as a zero reference. Subsequent readings are rotated relative to this reference using the Hamilton product with the conjugate of the reference quaternion:

Tare equation
qout = qref-1 · qin

where qref-1 = conjugate = (−i, −j, −k, +w) 
// Hamilton product: q_out = q_ref_inv * q_in
float ri = -g_ref_qi, rj = -g_ref_qj, rk = -g_ref_qk, rr = g_ref_qr;
*or_ = rr*qr - ri*qi - rj*qj - rk*qk;
*oi  = rr*qi + ri*qr + rj*qk - rk*qj;
*oj  = rr*qj - ri*qk + rj*qr + rk*qi;
*ok  = rr*qk + ri*qj - rj*qi + rk*qr;

2.4 Connection Loss Detection

The slave tracks consecutive ESP-NOW send failures via the send callback. After 20 consecutive failures (SEND_FAIL_THRESHOLD), the slave resets to IDLE and resumes beacon transmission. The master rediscovers it automatically.

if (status != ESP_NOW_SEND_SUCCESS) {
    g_consec_send_fails++;
    if (g_consec_send_fails >= 20) {
        g_streaming  = false;
        g_node_state = NODE_IDLE;  // auto-reset
    }
}

03 Time Synchronization

The system synchronizes three independent clocks: PC wall-clock, master local timer, and slave local timers. This enables absolute timestamps in CSV recordings and inter-node sync monitoring.

3.2 Equations

Step 1 — PC → Master: Bridge sends current epoch (ms) over serial.

PC sync command
sync_epoch_ms = int(time.time() × 1000)
serial.write(f"S{sync_epoch_ms}\n")

Step 2 — Master stores reference: On receiving the sync command, the master captures both the PC epoch and its own local clock.

Master sync storage
g_sync_epoch_ms = epoch_ms // PC wall-clock (uint64)
g_sync_master_us = esp_timer_get_time() // master local clock (µs)

Step 3 — Master stamps each frame: When forwarding a slave's IMU packet, the master appends its own time offset in milliseconds.

Master frame timestamp
ts_ms = (esp_timer_get_time() − g_sync_master_us) / 1000

Step 4 — Master → Slaves: Master broadcasts its current esp_timer_get_time() to all slaves via CMD_SYNC.

Slave clock offset
g_time_offset_us = (int32_t)(master_ts − slave_ts)

Step 5 — Slave timestamps: Each outgoing IMU packet carries the slave's local clock adjusted by the offset, aligning it with the master's timebase.

Slave timestamp generation
timestamp_us = (uint32_t)esp_timer_get_time() + (uint32_t)g_time_offset_us

Step 6 — Bridge reconstruction: The Python bridge reconstructs the absolute wall-clock epoch from the master's offset.

Absolute epoch reconstruction
absolute_ms = sync_epoch_ms + ts_offset_ms
🔎
Overflow note: PC epoch in 2026 ≈ 1.7 trillion ms, which overflows uint32_t. The master stores the full uint64_t epoch but only transmits a uint32_t offset in each frame. The bridge holds the full epoch and reconstructs: sync_epoch_ms + offset_ms.

3.3 Sync Error Measurement

The dashboard overlay monitors two metrics in real time:

Per-node latency — how stale the data is at the browser:

Pipeline latency
latency_ms = Date.now() − timestamp_ms

Inter-node sync error — the maximum clock delta between any two streaming nodes, compensated for packet arrival jitter:

Max inter-node delta
For each pair of nodes (i, j):

Δarrival = arrivali − arrivalj
Δtimestamp = timestamp_msi − timestamp_msj

errorij = |Δtimestamp − Δarrival|

max_delta = max(errorij) for all pairs

Threshold classification:

RangeBadgeMeaning
< 5 ms<5 msExcellent — well within real-time tolerance
5 – 10 ms5-10 msGood — acceptable for most applications
10 – 20 ms10-20 msWarning — noticeable drift, consider re-sync
> 20 ms>20 msBad — significant desync, re-sync required

⏰
Clock drift: ESP32 crystal oscillators drift at approximately ±10 ppm. At 10 ppm, two unsynchronized clocks diverge by ~0.6 ms per minute. Periodic re-sync (every 5 minutes) keeps inter-node error well under 5 ms.

CRC8 computation used for all frame integrity checks:

uint8_t crc8(const uint8_t *data, size_t len) {
    uint8_t crc = 0x00;
    for (size_t i = 0; i < len; i++) {
        crc ^= data[i];
        for (int b = 0; b < 8; b++)
            crc = (crc & 0x80) ? (crc << 1) ^ 0x07 : (crc << 1);
    }
    return crc;
}
